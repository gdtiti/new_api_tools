## 测试目标

- 需要被保护的核心行为:
  - 渠道健康度聚合结果的口径稳定：请求量、错误率、平均响应时间、健康分层。
  - 渠道并发估算算法在固定样例日志下可复现。
  - 订阅分析在“表存在 / 表不存在”两种情况下都给出正确响应。
  - 临时账号管理在“主库写入 + sidecar 写入”双写场景下保持一致或显式失败。
- 这轮必须覆盖的失败模式:
  - 大时间窗聚合导致慢查询或缓存失效。
  - 把“并发估算”误实现成“实时精确值”或口径漂移。
  - 订阅表缺失时静默返回空数据。
  - 临时账号 sidecar 写入失败但主库未回滚。

## 单元测试设计

### UT-001

- 目标:
  - 校验渠道健康度聚合逻辑。
- 覆盖函数/模块:
  - `channel_insights_service`（待新增）
- 输入:
  - 固定窗口内的渠道日志样例、渠道当前状态样例。
- 预期输出:
  - 每个渠道的请求量、错误率、平均响应时间、健康分层与预期一致。
- 失败表现:
  - 聚合字段错位、健康等级错误、窗口边界错误。

### UT-002

- 目标:
  - 校验渠道并发估算逻辑。
- 覆盖函数/模块:
  - `channel_insights_service` 中的并发估算函数（待新增）
- 输入:
  - 一组带 `channel_id / created_at / response_time` 的日志样例。
- 预期输出:
  - 指定渠道的峰值并发与预先手算结果一致。
- 失败表现:
  - 对重叠区间计算错误；单位换算错误。

### UT-003

- 目标:
  - 校验订阅分析的 schema 探测与 unsupported 分支。
- 覆盖函数/模块:
  - `subscription_analytics_service`（待新增）
- 输入:
  - 有订阅表的模拟元数据；无订阅表的模拟元数据。
- 预期输出:
  - 表存在时返回统计结构；缺表时返回显式 unsupported。
- 失败表现:
  - 缺表时返回空数组或伪成功。

### UT-004

- 目标:
  - 校验临时账号双写一致性。
- 覆盖函数/模块:
  - `temporary_account_service`（待新增）
- 输入:
  - 正常创建请求；sidecar 写入失败场景。
- 预期输出:
  - 正常场景主库/sidecar 均写入成功；
  - 失败场景显式回滚或标记失败。
- 失败表现:
  - 主库与 sidecar 状态分裂且未报警。

## 属性测试设计

### PT-001

- 性质:
  - 渠道并发估算对同一日志集合的结果应与输入顺序无关。
- 输入空间:
  - 相同日志集合的不同排列顺序。
- 不变量:
  - 峰值并发、窗口内总请求数保持一致。
- 预期失败模式:
  - 估算逻辑依赖输入顺序，导致结果漂移。

### PT-002

- 性质:
  - 订阅 schema 探测必须是显式二值结果：支持或不支持，不能默默退化成空统计。
- 输入空间:
  - 存在、部分存在、完全不存在的表结构组合。
- 不变量:
  - 缺少关键表时返回 unsupported 标识。
- 预期失败模式:
  - 关键表缺失时仍返回 success=true 且 data 为空。

## 测试夹具与数据策略

- 夹具:
  - 渠道日志最小样例集
  - 订阅 schema 探测 mock
  - 临时账号双写 mock / 测试数据库
- 真实数据策略:
  - execution 阶段优先用本地真实 schema 或目标库只读探测结果校验；
  - 若使用样例日志，必须由真实字段抽象而来，不得凭空造结构。
- 禁止使用的假路径:
  - 禁止在缺少订阅表时返回“空列表即视为无订阅”；
  - 禁止用前端硬编码示例数据充当统计结果；
  - 禁止用不进入主库写路径的“伪临时账号”冒充真实创建成功。
