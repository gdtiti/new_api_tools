## 术语与定义

- 渠道健康度：面向管理后台的综合观察指标，至少包含状态、请求量、错误率、响应时延、最近测试时间。
- 渠道并发跟踪分析：基于日志和响应时长推导的并发估算或热点分析，不等于 relay 调度器的实时限流。
- 订阅统计：套餐层面的数量、活跃状态、使用情况与在用明细分析。
- 临时账号：为短期使用而创建并需要单独识别、管理、回收的账号；在当前项目推荐用 sidecar 元数据识别，而不是先强改主库 schema。
- sidecar 元数据：当前项目本地 SQLite/缓存层中维护的扩展业务元数据，用于补足主库没有的中间件专属信息。

## 调研结论

- 结论 1:
  - 当前项目做“渠道健康度分析”是低风险增量，因为已有 `channels` 概览、`logs` 聚合、缓存预热和分析服务框架。
- 结论 2:
  - 当前项目无法像 `new-api-gdtiti` 那样在调度链路内做“真实并发限制”，但能基于 `logs.channel_id` 和响应时长做“并发跟踪分析”。
- 结论 3:
  - 订阅分析在当前项目中属于全新增量模块，前提是目标主库存在订阅相关表；否则必须显式报依赖缺失。
- 结论 4:
  - 临时账号管理最适合采用“主库标准用户/token + 本地 sidecar 元数据”的方案，这比直接复制参考项目的主库 schema 更贴合当前项目定位。
- 结论 5:
  - 当前项目 wiki 的项目概览已过期，后续 execution 前应先把“FastAPI 中间件”这一实现事实同步回知识层。
